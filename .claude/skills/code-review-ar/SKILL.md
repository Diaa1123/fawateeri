# مراجعة الكود — Code Review بمعايير عربية

## متى تُستخدم هذه المهارة
- عند طلب "راجع الكود" أو "review"
- بعد إنشاء ميزة جديدة كاملة
- قبل النشر أو الـ deployment
- عند ظهور مشاكل غير واضحة

## قائمة الفحص — تُنفذ بالترتيب

### 1. فحص RTL والعربية
```
□ هل كل الاتجاهات تستخدم logical properties؟ (ps/pe بدل pl/pr)
□ هل الأيقونات الاتجاهية تنعكس؟ (rtl:rotate-180)
□ هل النصوص المعروضة للمستخدم بالعربي؟
□ هل رسائل الخطأ بالعربي؟
□ هل الأرقام تُعرض بصيغة عربية؟ (Intl.NumberFormat)
□ هل line-height مناسب للنص العربي؟ (1.7+)
□ هل الخط يدعم العربي بشكل صحيح؟
```

### 2. فحص الأمان
```
□ هل كل input معمول له validation بـ Zod؟
□ هل الـ protected routes تتحقق من الجلسة؟
□ هل فيه SQL injection ممكن؟ (المفروض لا مع Prisma)
□ هل البيانات الحساسة محمية؟ (كلمات مرور، tokens)
□ هل الـ API يتحقق من الصلاحيات؟
□ هل فيه rate limiting على APIs الحساسة؟
□ هل الـ .env مضاف في .gitignore؟
```

### 3. فحص الأداء
```
□ هل الصور تستخدم next/image مع sizes؟
□ هل فيه N+1 queries في Prisma؟ (include بدل queries متكررة)
□ هل البيانات الكبيرة فيها pagination؟
□ هل Client Components محدودة بالحاجة فقط؟
□ هل فيه re-renders غير ضرورية؟
□ هل الـ imports dynamic عند الحاجة؟ (next/dynamic)
```

### 4. فحص جودة الكود
```
□ هل فيه تكرار ممكن استخراجه لـ utility أو component؟
□ هل التسمية واضحة ومتسقة؟
□ هل الملفات في المجلدات الصحيحة؟
□ هل فيه console.log منسية؟
□ هل الـ TypeScript types محددة؟ (بدون any)
□ هل الـ error handling موجود وواضح؟
□ هل فيه TODO أو FIXME تحتاج معالجة؟
```

### 5. فحص تجربة المستخدم
```
□ هل أزرار اللمس بحجم مناسب؟ (44px minimum)
□ هل فيه loading states؟ (skeleton، ليس spinner فقط)
□ هل فيه empty states؟ (رسالة لما ما فيه بيانات)
□ هل فيه error states؟ (ماذا يحصل لو فشل الطلب)
□ هل النماذج تعطي feedback فوري؟
□ هل الأزرار تتعطل أثناء الإرسال؟ (prevent double submit)
```

## تنسيق التقرير

عند عرض نتائج المراجعة، استخدم هذا التنسيق:

```markdown
## نتيجة المراجعة

### 🔴 مشاكل حرجة (يجب الإصلاح)
- [وصف المشكلة] — الملف: `path/to/file.tsx` سطر XX
  **الإصلاح**: [الحل المقترح]

### 🟡 تحسينات مهمة (يُفضل الإصلاح)
- [وصف التحسين]
  **الاقتراح**: [كيف يتحسن]

### 🟢 ملاحظات بسيطة
- [ملاحظة]

### ✅ أمور جيدة
- [ما تم تنفيذه بشكل صحيح]
```

## الممنوعات في المراجعة
- ❌ لا تتجاهل مشاكل RTL — هذه أولوية قصوى
- ❌ لا تقول "الكود جيد" بدون فحص فعلي
- ❌ لا تقترح إعادة بناء كاملة لمشكلة بسيطة
- ❌ لا تتجاهل رسائل الخطأ بالإنجليزي المعروضة للمستخدم
