## 2. التقنيات

### Frontend
- **Framework**: Next.js 16.2.2 (App Router)
- **Language**: TypeScript 6.0.2 (strict mode)
- **Styling**: Tailwind CSS 3.4.17 + PostCSS 8.4.49 + Autoprefixer 10.4.20 (داكن، بدون أنيميشن)
- **Charts**: Recharts 3.8.1
- **Data Fetching**: TanStack Query 5.96.2
- **Forms**: React Hook Form 7.72.1 + Zod 4.3.6
- **Icons**: Lucide React 1.7.0

### Backend
- **API**: Next.js Route Handlers (REST)
- **Auth**: Jose 6.2.2 (JWT) + bcryptjs 3.0.3
- **Database**: Airtable REST API

### خدمات خارجية

| الخدمة | الغرض | الحالة |
|--------|-------|--------|
| Airtable | قاعدة البيانات (فواتير + مستخدمين) | يحتاج إعداد |
| Make.com | جمع الفواتير من الإيميلات + إرسال webhook | يحتاج إعداد |
| Google Drive API | تخزين ملفات PDF + تحويلها لنص | يحتاج إعداد |
| OpenAI API (GPT-4o) | تحليل محتوى الفواتير واستخراج البيانات | يحتاج إعداد |

## 5. ملاحظات مهمة

### قرارات مبكرة
- اخترنا Airtable بدل PostgreSQL لأن المشروع داخلي صغير ويحتاج مرونة بدون migrations
- اخترنا JWT (Jose) بدل NextAuth لأن المستخدمين محدودين والأدمن يضيفهم يدوياً
- اخترنا Route Handlers بدل tRPC لأن الـ API بسيط ولا يحتاج complexity إضافي
- اخترنا بناء UI components يدوياً بدل shadcn/ui لأن المشروع صغير ومحدود الصفحات
- التصميم داكن بدون أنيميشن — مستوحى من UI UX Pro Max

### مخاطر معروفة
- Airtable API rate limit (5 requests/sec) — نحتاج caching مع TanStack Query
- Google Drive API يحتاج service account ومفاتيح
- OpenAI API costs — التحليل الذكي يكلف per request (GPT-4o)
- Make.com webhook security — نحتاج secret key للتحقق

### خارج النطاق (مؤجل)
- تطبيق جوال أصلي (React Native / Flutter)
- ربط مع نظام محاسبة خارجي (Zoho / QuickBooks)
- سير عمل موافقات (Approval Workflow)
- إشعارات بريدية أو واتساب
- تقارير PDF قابلة للتحميل
- نظام تصنيف/تاغ للفواتير
